# SAGE Persona Clustering - 实验日志 (Experiment Log)

## 2026-03-11: 网格搜索初步分析 (Grid Search Initial Analysis)

### 实验现象记录 (Observations)
1. **推土机距离轮廓系数 (EMD Silhouette Score) 为负数**:
   - 分数范围在 `-0.4` 到 `-0.5` 之间。
   - 随着 **Persona 数量 (N)** 的增加，轮廓系数持续降低（变得更负）。
   - 与 **迭代次数 (Iter)** 的关联呈现非单调性，可能存在极值。
2. **困惑度 (Perplexity)**:
   - 随着 **Persona 数量 (N)** 的增加而降低。
   - 随着 **迭代次数 (Iter)** 的增加呈单调降低趋势。

---

### 技术分析与诊断 (Technical Analysis & Diagnosis)

#### 1. 为什么轮廓系数为负？
在聚类评估中，负的轮廓系数意味着：**样本到所属类簇中心的距离，大于到最近邻类簇中心的距离。** 
针对本项目，可能的原因如下：
- **特征空间重叠 (Feature Space Overlap)**: 文学人物的角色特征（Agent/Patient 等）具有极高的共性。大多数人物都共有一些“核心动作”（如 see, know, go），导致人物在词分布空间中高度重叠，边界极不明显。
- **EMD 地面距离 (Ground Distance) 的敏感性**: 我们的地面距离是基于 Word2Vec 聚类中心计算的。如果 Token Clusters 之间的语义距离（Euclidean）不能很好地代表人物角色的语义逻辑，Sinkhorn 距离就会引入大量噪声。
- **高维稀疏性**: 虽然我们映射到了 1000 个词簇，但每个角色的有效特征依然稀疏。在 Sinkhorn 迭代中，稀疏分布的传输成本计算容易导致质心向全局均值靠拢，从而失去了区分度。

#### 2. 指标趋势的含义
- **Perplexity 的单调下降**: 
  - 这说明模型在**统计意义上**是收敛的。更多的 Persona 提供了更强的参数拟合能力（Capacity），而更多的 Iteration 确保了 MLE/EM 过程找到了局部最优解。
  - **结论**: 模型的概率预测能力没有问题，它确实学到了数据中的统计规律。
- **Silhouette 的下降**:
  - 这反映了**语义分离度**的恶化。随着 N 增加，模型被迫在已经非常拥挤的语义空间中进一步切割类簇，导致类内差异增大，类间重叠更加严重。
  - **建议**: N=4 或 N=8 可能是语义上的合理区间，过大的 N 会导致严重的“人物碎裂化”。

#### 3. 潜在的优化方向
- **重新审视 Token Clusters**: 检查 `data/processed/word2vec_clusters.csv`。如果聚类质量不高，地面距离矩阵 $M$ 就是错误的。
- **引入背景词过滤**: 文中提到的 $eta_{bg}$ 应该吸收大部分共性词。如果背景偏置项没有起到足够的“抽干”作用，Persona 特征就会被共性词淹没，导致所有 Persona 看起来都差不多。
- **正则化强度**: 目前 L1 Lambda 为 `1e-6`，这非常弱。可以尝试增加 Lambda (如 `1e-4` 或 `1e-3`)，强制模型学习更稀疏、更具代表性的 Persona 特征，从而提高类间区分度。
